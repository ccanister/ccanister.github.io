防抖和节流

发表于 2019-11-07 | 分类于 javaScript

最近在用rxjs时候对于throttleTime和debounceTime这两个操作符很困惑，用起来就是瞎蒙，哪个看起来对就用哪个。之前就有接触过throttle，用于控制input输入框输入频率。于是特地去搜索一哈节流是啥玩意。

概念

节流限制了随时间调用函数的次数。就像每100毫秒最多执行一次函数。

假设在正常情况下，你在10s内调用这个函数1000次。但是你限制100ms调用一次，那么最多调用100次。

(10s * 1,000) = 10,000ms
10,000ms / 100ms throttling = 100 maximum calls

防抖限制了函数在特定时间内没用调用后再去触发。就像100毫秒内没有调用这个函数再去触发。

可能函数会在忽然在3s内被调用1000次。如果你做了防抖并将频率限制在100ms，那么这个函数只会在3.1s时候被真正触发。每当这个函数被调用时都会重置其内部定时器。

重点是什么

这两个行为非常经典的场景就是对DOM操作，比如滚动进度条或者对调整窗口大小。比如，对一个元素绑定了滚动事件，并往下滚了5000px，那么触发事件的频率是很高的。如果绑定的事件中做了大量计算或者操作DOM等事，就会发现有性能影响。如果可以尽量少的执行函数，那么在体验上没有太多卡顿，这是值得的。

几个经典例子：

  1. 等用户停止拖拽窗口
  2. 等到用户停止输入再进行ajax请求
  3. 测量页面的滚动位置，最多50ms后再响应
  4. 在页面上拖组件时为了得到更好的性能
