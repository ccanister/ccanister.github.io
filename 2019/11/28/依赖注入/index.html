依赖注入

发表于 2019-11-28

什么是依赖注入

在面向对象语言中，在调用某个类的时候，必选先实例化该类，生成一个对象。

如果这个类需要调用其他类的时候，甚至其他类也依赖更多的其他类，我们就需要无数次的实例化。

而依赖注入，即依赖注入到容器（IOC）中，它里面有某个类和其依赖，当你需要实例时候，他会把你实例化好给你。

依赖注入的实现原理

需要实现依赖注入，最主要的是我们需要知道这个类的依赖，即构造器中的参数类型。

通过reflect-metadata可以获得该类的一些信息。

  • 方法参数类型(design:paramtypes)，返回类型(design:returntype)
  • 成员类型(design:type)
  • 自定义信息(defineMetadata)

Angular内部如何实现依赖注入

Angular在v5时候舍弃了ReflectiveInjector，因为其代码比较多，从而引入了StaticInjector，而这也是在注册AppModule使用的，对于组件层使用了Injector_。StaticInjector中不会出现和reflect有关代码，因为需要获得构造器参数代码转移到了compile层面，可以看到compiler/src/metadata_resolver文件中的 _getDependenciesMetadata方法：

获取参数逻辑在this._reflector.parameters(typeOrFunc)方法中，可以取得该参数的依赖和注解。注解就是我们附加上去的@Host、@Self等修饰器。如果我们的参数类型没有明确指定或者不写，就会抛出Can’t resolve all parameters for xxx异常。

Angular依赖注入使用

  1. ElementRef: 注入自身 DOM 元素，在指令中用的较多。
  2. 利用抽象类，拿到父组件实例。
  3. 如果类发生了引用循环，也可以使用forwardRef或者Injector。
  4. 由于是单例的，很适合做状态存储或者父子组件传递数据。
  5. 可以模仿mvc结构，抽出service和api层，而component只有数据和视图。
