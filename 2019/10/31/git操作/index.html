git操作[持续更新]

  • git clone xxx (new directory)这样克隆下来的时候就会重命名文件夹

  • 分支操作

    • git branch -a 查看远程和本地分支

  • git branch -r 查看远程分支

    • 删除了远程分支后，本地仍能看到该分支，可以使用git remote prune origin命令。

  • 比较

    • 一般直接使用git diff可以查看当前修改文件内容，
      • git diff -name—only查看修改文件名称
    • git diff branch1 brnahc2查看不同分支文件内容差异
      • git diff diff branch1 branch2 —stat查看不同分支差异文件名称

有了堡垒机之后，需要传送文件。

通过nc命令

  • 接收文件端： nc -l 1234（端口） > dist.tar（文件重命名）
  • 发送文件端： nc 172.17.168.1 1234 < dist.tar（需要传输的文件）

打通免密

因为是docker虚拟机，我们公司账号都是admin登录，所有没法使用网上的ssh-copy-id -i id_rsa.pub xxx，全部都是手动完成。（当前目录都是/home/admin）

  • spiro1主机生成ssh秘钥：ssh-keygen -t rsa -P ''。

  • 将.ssh下的.id_rsa.pub拷贝到spiro2主机下的.ssh/authorized_keys文件夹下。

  • 如果.ssh/authorized_keys不存在，就要创建该目录。还要去修改/etc/ssh/sshd_config文件配置。

  • 修改服务后，重启该服务：systemctl restart sshd.service。

  • 这时候如果说ssh仍然需要密码，那么就要看下authorized_keys文件配置。这个坑踩了很久，说是只有文件所有者才有写的权限，如果其他用户也有的话，就会拒绝读取该文件。所以需要给文件价格权限：chmod 644 /home/admin/.ssh/authorized_keys。
