new内部原理

Brendan Eich想让Javascript向传统的面向对象编程语言一样，比如C++、Java等。在那些语言中，我们都会对类使用new操作符来实例化。因此他也在Javascript中采用了new操作符。

  • C++有构造函数，用来初始化对象的属性。因此new后面跟着的是函数。
  • 我们需要吧方法统一放到对象的某个地方。因为我们使用的是原型语言，那我们放到property属性。

new操作符接受一个F函数和形参：new F(arguments...)。它做了一下三步：

  • 创建类的实例。他是一个空对象除了__proto__属性指向F.property
  • 实例化对象。函数F利用传过来的参数比调用，this指向该实例。
  • 返回该实例

配置文件

  • entry：可以是字符串（单一入口），也可以是Object（多入口）。object中的key可以在输出中起到作用。格式如下：
  • output：转换输出文件的规则。用的较多的有：
    • path：输出路径，一般是dist文件夹，即path.resolve(__dirname, 'dist')
    • filename：对应的输出文件名，有几种变量替换。
      • [name]：即之前说的entry入口的key
      • [hash]：本次编译的hash版本，会将所有输出文件替换成同一个变量，即全局替换。
      • [chunkhash]：一个文件代表一个chunk，这样不同的文件就有不同的hash。如果两次编译，文件未发生任何变化，hash也不会发生改变。
  • module：webpack只能对js文件打包。如果想要打包例如css、img这些文件，就需要使用loader类似插件进行预处理打包。一般只使用rules参数，而rule参数又有几个自参数：
    • test：指需要匹配哪些资源。
    • loader/use：代表用了哪些loader进行处理，如果只有一个，就是用loader，有多个的话，以数组的形式，按照处理的顺序从下到上。
    • options：传递给loader的参数。
    • exclude/include：即处理的目录和忽略处理的目录。
  • plugin：辅助于webpack，会在整个构建过程中起到作用。

基本类型

  • null
  • undefined
  • number
  • string

  • Symbol
    • es6新增类型
    • var s = Symbol()生成值，可接受任意值作为参数。如果用这种方式生成，那么Symbol(1) === Symbol(1)为false。其产生的值每个都是独一无二的。但是Symbol.for(1) === Symbol(1)为true。可类比为Java的intern()方法，如果全局中，已存在该变量那么直接返回该变量。
    • Object.keys()和Object.getOwnPropertyNames()不会返回包含Symbol的属性名，但有Object.getOwnpropertySymbols()方法可以返回。

对象

    • 基本类型，除了null会显示Object，其他都可以显示正确的类型。
    • 引用类型，只能显示Object
  • Object.prototype.toString.call(xx)可获得类似[object Type(如Array)]返回值
  • instanceof可返回Type。

  • +运算符：如果都是数字，那么直接相加。否则转换成字符串类型再相加。

侦查检测步骤

​ 如果你像我一样想对ANgular的变化检测有全面的了解，由于网上灭有太多有用的信息，你必须深究源码。大部分文章只是提交到每个组件有自己的 变化检测器负责检测组件，但是他们大部分时候关注于不变的例子和变化检测策略。这篇文章将会让我们理解为什么使用不变的例子和如如何改变变化检测策略影响检查。当然，你也会从这篇文章中想到不同的情景用来优化性能。

​ 这篇文章包括两部分，第一部分探讨大量技术细节，包含了源码的链接。它解释了在底层变化检测工作原理。这些内容基于Angular 4.0.1。这和早期的2.4.1的变化检测原理是不一样的。如果对这方面也感兴趣，可以阅读这篇答案。第二部分展示了如何在应用中改变检测机制，这部分内容对于4.0和2.4版本均适用。

核心概念-视图

​ 在教程中提到过，Angular应用是一颗组件数。但是，在底层Angular使用了更低一级的抽象-view。view和component有着直接的关系：每个view关联着一个component，反之亦然。view中有个componet变量，其中持有其相关联的组件类实例。所有操作，比如检查属性、DOM更新都是在view上操作的。所以更专业的说法是，Angular应用是view树，而组件可以看做是view更高级的抽象。在源码中，view是这么描述的：

​ 在这篇文章中，我会交替使用组件 view和组件这两个概念。

​ 每个view都可以通过nodes属性链接到子view，因此可以直接操作view。

view状态

​ 每个view都会有状态，Angular会根据状态判断检测或忽略view和其子view。view的状态比较多，但和本文相关的只有以下几个状态：

1. FirstCheck
2. ChecksEnabled
3. Errored
4. Destoryed

​ 如果ChecksEnabled是false或者视图处于Errored或者Destoryed状态，那么该视图和子视图都会跳过视图检测。默认情况下，所以视图会被初始化为ChecksEnabled状态，除非被设置为ChangeDetectionStrategy.OnPush。这将会进行详细的讨论。状态可以同事拥有，比如视图可以同时被设置为FirstCheck和ChecksEnabled。

​ Angular有较高等级抽象方法去操作视图。我曾写过这篇文章谈到过。其中有个概念是ViewRef。它包装了底层视图组件，以及有操作方法detectChanges。当有异步事件发生时，Angular在最顶层抽象上ViewRef上触发变化检测。当它检测完自身组件后，会再去检测子组件。

​ 你可以使用ChangeDetectorRef已用将viewRef注入到组件的构造器中。

​ 查看其类定义:

变化检测策略

​ 视图的变化检测的主要逻辑在checkAndUpdateView方法中。大部分功能是对其子组件视图操作。这个方法从主组件中开始递归调用。这意味着当地归属展开后，子视图会成为父节点在方法下次调用时。当方法被调用后，视图会按照以下顺序更新检查：

1. 如果`view`是第一次检查，设置`ViewState.firstCheck`为`true`，如果之前已经检查过，那么设置为`false`。

  1. 检查和更新子组件和指令的input属性

  2. 更新子视图更新检测状态（部分子组件检测策略改变）

  3. 对视图的组件进行更新检测（重复列表操作）

  4. 如果子组件绑定了OnChanges方法，那么调用它。

  5. 子组件调用OnInit和ngDoCheck方法（OnInit方法只在初始化时候调用）

  6. ？？

  7. 子组件调用AfterContentInit和AfterContentChecked生命周期方法（AfterContentInit方法只在第一次检查时调用）

  8. 如果当前视图中，有插入的视图组件发生变化，就去更新。

  9. 视图的子组件更新检测（重复列表）

  10. ？？

  11. 子组件实例调用AfterViewInit和AfterViewChecked生命周期方法（AfterViewInit只在对此检查时调用）

  12. 当前视图禁止检查（部分检测策略改变）

这里只有部分需要关注。

​ 第一，子组件的OnChanges方法会在子视图检查前调用，即使子视图的更新检测设置成跳过也会调用。这点是非常重要，在第二节中我们会看到如何运用这个知识。

​ 第二，当视图被检查时候，视图中DOM的更新也是作为更新检测原理一部分。这意味这如果组件不被检查，即使组件属性在模板中改变，DOM也不会更新。第一次检查前模板就会渲染完成。我提及的DOM更新指的是插值更新。所以当有如下模板<span>some </span>时，第一次检查前span标签已经渲染好了，检查时候，只有会重新被渲染。

​ 另一件有趣的事是在更新检测时候，子组件的状态可以被改变。我之前提过，所有组件默认初始化为ChecksEnabled状态。但是如果组件使用了OnPush策略，在第一次检查后，更新检测会被禁止。（上述第9步）

​ 这意味着当更新检测时候，该组件和子组件的检查都会被忽略。文档中关于OnPush的策略解释为组件只有在绑定的值变化时候才会更新。

Angular管道

简介

管道这个东西用的不多，现在在项目里用的也只有DatePipe来转化时间日期，今天重新看了哈官方文档，发现这个东西还是蛮好用的。主要用来做值-显示，比如浮点数只显示一位小数，可以这么写：3.222 | number(1.1-1)，将被转化为3.2。这个参数的意思是：小数点前至少显示1位，小数点后至少1位，至多1位，其实只显示1位。

内置管道

官方的内置管道，用的比较多的有：

  • JsonPipe用来转换json字符串，调试很方便。

  • DatePipe转换任意格式字符串

  • DecimalPipe转化任意格式的字符串，PercentPipe转化成百分比字符串

  • KeyValuePipe将对象或Map转换成key-value格式，可在模板中直接调用

  • LowerCasePipe、UpperCasePipe、TitleCasePipe都是转换大小写，最后一个意思只将字符串的第一个字母转成大写，其余都是小写，单词可以任意分隔符分割

  • SlicePipe字面意思，切割字符串或数组

  • SanycPipe会将Observer对象数据取出
